编译器与解释器的区别：

1. 编译器是将高级语言转化为机器码，存储在硬盘里，最后程序的运行是通过用户手动调用进行；
2. 解释器是将高级语言直接执行，它并不会生成一个可执行文件。

结合实际：

C 语言就是一个典型的编译型语言，我们使用 GCC 编译器将其转化为机器码 `.o` 文件，最后执行；
而 Python 从直觉上来讲是个解释型语言，使用的是 CPython 解释器，但是 Python 内部其实也有做一些编译工作（将代码转化为字节码 `.pyc`）。

现在，咱们探索一下 Vue 编译器的扫描器和分析器是手写的还是通过 Compiler-Compiler 工具生成的。
进入 [Vue3源码](https://github.com/vuejs/core/blob/main/packages/compiler-core/src/) 可以看到，所有东西基本都是手搓实现出来的，扫描器的实现在 `tokenizer.ts`，解析器的实现在 `parser.ts`。
Vue3 的扫描器定义了在接收到不同的 token 之后应该转化到的状态 `State` 以及表示转换过程的回调函数 `cbs`，使用的是有限状态机的思想（编译完备的语言用的都是有限状态机的思想）；解析器根据扫描器进行分词，然后组合生成 AST，在前端中也就是一个挂载着许多子节点的标签。
新的认识：某些特定的标记过的标签（渐进式框架的原因就是因为我们在导入 Vue3 的包之后，其编译器会将挂载的 DOM 节点进行编译）会被 Vue3 的编译器（通过 TS 语言实现，编译为了 JS 代码）在 JS 环境下进行编译，得到最终浏览器需要解释执行的 JS 代码。

对于昨天遗留的问题：
1. jlox 是一个纯解释器，代码立即执行，不会生成可执行文件。学习内容大多数是一些基本的比如变量、表达式。其中最核心的一个概念就是递归下降解析的方式。递归下降解析可以简单地理解为：对于一个巨大的表达式，我们首先找到一个合适（优先级较低）的运算符号，通过运算符号将表达式分为两个部分，形成一个树的结构。接着再对左右两个部分执行相同的操作，循环往复，直到分裂为叶子节点，然后计算结果。
2. 现在开始进行 clox 的学习

clox 首先使用了一个概念叫做虚拟机。

虚拟机是什么呢？

可以理解为一个虚拟的平台，具有虚拟的架构和芯片。有了虚拟机，我们无需在意我们现在所使用的机器的架构和芯片类型，从而能够做到 lox 语言代码的可移植性（在大多数平台上都能运行，因为虚拟机能够通过编写语言 Rust、C 等进行跨平台编译）。

clox 通过编译器前端将代码变成一种中间形式 IR（字节码）。接着，要运行的时候将 IR 载入虚拟机进行执行。（所以 clox 是解释型与编译型的结合，类似 Java 语言）

目前的理解：我们的编译前端需要对虚拟机进行适配，生成虚拟机能够执行的 IR。（因为书籍先写的 VM，肯定是后者要适配前者）
