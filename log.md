编译器与解释器的区别：

1. 编译器是将高级语言转化为机器码，存储在硬盘里，最后程序的运行是通过用户手动调用进行；
2. 解释器是将高级语言直接执行，它并不会生成一个可执行文件。

结合实际：

C 语言就是一个典型的编译型语言，我们使用 GCC 编译器将其转化为机器码 `.o` 文件，最后执行；
而 Python 从直觉上来讲是个解释型语言，使用的是 CPython 解释器，但是 Python 内部其实也有做一些编译工作（将代码转化为字节码 `.pyc`）。

现在，咱们探索一下 Vue 编译器的扫描器和分析器是手写的还是通过 Compiler-Compiler 工具生成的。

进入 [Vue3源码](https://github.com/vuejs/core/blob/main/packages/compiler-core/src/) 可以看到，所有东西基本都是手搓实现出来的，扫描器的实现在 `tokenizer.ts`，解析器的实现在 `parser.ts`。

Vue3 的扫描器定义了在接收到不同的 token 之后应该转化到的状态 `State` 以及表示转换过程的回调函数 `cbs`，使用的是有限状态机的思想（编译完备的语言用的都是有限状态机的思想）；解析器根据扫描器进行分词，然后组合生成 AST，在前端中也就是一个挂载着许多子节点的标签。

说到虚拟机，先想想为什么要有虚拟机这个概念？

在编译原理中，我们通过编译器将高级语言变成机器能够执行的机器码。但是不同的电脑（架构、芯片）需要的机器码格式是不同的。比较直观的例子就是不同平台的可执行文件格式不同： Windows 的 exe、Linux 的 elf。其中一个平台的可执行文件搬到另一个平台肯定是不能运行的。

对于这种不同平台不同机器码格式的情况，我们要如何做到我们写的代码能够一端编译多端运行呢？

比较麻烦的方式是在每个平台用相应的编译器编译然后执行，或者搞些定制化的高级操作。

但这些方式都太麻烦了，所以我们可以引入虚拟机！

引入虚拟机之后，我们可以让高级语言编译的产物变成虚拟机能够执行的内容，也就是字节码。然后在平台的虚拟机中运行这些字节码。

不过这里有很重要的一点，撰写虚拟机的代码需要是能够跨平台的语言，因为这样我们才能够吃到跨平台的福利，让虚拟机直接编译为不同平台的可执行文件进行分发。否则你还要面临重复的问题：不同平台下虚拟机的代码兼容吗？

总结，虚拟机就像是计算机系统中一个抽象出来的软件接口。高级语言只需要将编译产物变为虚拟机能够执行的代码即可，平台适配的问题交给虚拟机（或者说交给跨平台语言）。

## 2026.03.07

（早上）1h

又了解到了新的概念：虚拟机有最少两种形式。第一种是基于堆栈的虚拟机；第二种是基于寄存器的虚拟机。

寄存器虚拟机在性能上比堆栈虚拟机更快，为什么呢？

堆栈虚拟机在对表达式进行求值的时候需要反复进行入栈出栈的操作；而寄存器虚拟机对中间产物的使用更加方便，少了一次入栈操作（减少与内存的交互次数，要知道，CPU的计算速度可比内存读写快多了）。

另外寄存器虚拟机还拥有随机访问的权限，而堆栈虚拟机只能访问栈顶元素（消除数据重排的开销）。

最后一点（问ai的）：寄存器虚拟机更容易被 JIT 编译器优化，也就是我们这个项目待会要写的前端。为什么更容易被优化？因为寄存器虚拟机的逻辑与 CPU 的底层逻辑类似，直接转换比较容易；堆栈虚拟机需要将内容都读一遍分析出转换逻辑。
